# Review: stack and architecture

**Date**: 2026-07-17  
**Scope**: uncommitted stack scaffold on `main`  
**Reviewer**: Codex GPT 5, same model review by request

## Verdict

Approve with nits.

The monorepo builds, unit tests pass, the NestJS route passes its integration test, and the Playwright browser tests prove the public page and API proxy. The review did not find a blocker or major code issue.

## Minor findings

1. `test-preferences.json` selects Vitest for the repository, but `apps/api` uses Jest. A future `/test` run should select the test runner from the workspace package rather than relying only on the root preference file. This is a tooling accuracy issue, not a current test failure.

2. `playwright.config.ts` always allows an existing server on ports 3000 and 3001. A local browser run can therefore use a server started from a different working tree. This is convenient during development, but the future CI setup should start isolated servers so browser results always match the checked code.

## Strengths

1. The web rewrite has both configuration tests and a real browser proof.

2. The API has unit and integration coverage, and the root commands make the normal test paths easy to run.

3. The same origin `/api` boundary is working as designed, which avoids a browser CORS dependency during the scaffold stage.
