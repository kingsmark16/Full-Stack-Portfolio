# Launch journey

This journey is complete when the portfolio can be discovered, measured, monitored, and operated as a dependable production site within a small monthly budget.

### 19. Search and social visibility · needs a decision

Make every public page understandable to search engines and attractive when shared with recruiters or clients.
**Done when:** public pages have correct titles, descriptions, canonical addresses, structured data, sitemap entries, and social sharing images; private dashboard routes are excluded from discovery.

- [ ] Design it (spec): `/architect search and social visibility`

### 20. Analytics and error monitoring · needs a decision

Measure qualified contact conversion and site health without collecting more visitor data than needed.
**Done when:** visits, important project views, resume downloads, contact starts, and successful contact submissions are measured; public and dashboard failures reach error monitoring; consent and privacy behavior match the chosen tracking requirements.

- [ ] Design it (spec): `/architect analytics and error monitoring`

### 21. Production release · needs a decision

Release the public site, private dashboard, API, stored data, uploads, and notifications as one operable product.
**Done when:** production addresses use secure connections, configuration and secrets are separated from code, data changes and recovery are safe, public pages pass Core Web Vitals under normal traffic, WCAG 2.2 AA checks pass, health checks and logs support diagnosis, and the complete visitor and owner journeys pass on production.

- [ ] Design it (spec): `/architect production release`
